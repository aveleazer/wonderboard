# Wonderboard

7 AI सलाहकार जो आपसे भी असहमत हैं, और एक-दूसरे से भी।

> *"एकमात्र टूल जो वो बात कहेगा जो आप सुनना नहीं चाहते।"*

## समस्या

आप AI से कोई सवाल पूछते हैं। एक जवाब मिलता है। विनम्र, संतुलित, औसत-सा। कोई नहीं कहता "यह फ़ेल होगा।" कोई नहीं पूछता "आप यह कर ही क्यों रहे हैं?" कोई वो पागलपन भरा विकल्प नहीं सुझाता जो आपके दिमाग में आया ही नहीं।

असल ज़िंदगी में सबसे अच्छे फ़ैसले अलग-अलग सोच रखने वालों की बहस से निकलते हैं। Wonderboard वही बहस खड़ी करता है।

## सिर्फ़ Claude से पूछने से यह अलग कैसे है

**अलगाव।** हर सलाहकार स्वतंत्र रूप से सोचता है, एक अलग Claude कॉल में। जब एक ही प्रॉम्प्ट से संशयवादी और दूरदर्शी दोनों बनने की कोशिश होती है — तो घालमेल निकलता है। जब वे अलग-अलग हों — तो असली असहमति सामने आती है।

**दो राउंड।** राउंड 1 में सलाहकार एक-दूसरे की राय देखे बिना जवाब देते हैं। राउंड 2 में वे सबके जवाब पढ़ते हैं, बहस करते हैं, और अपनी राय बदलते हैं। व्यावहारिक विशेषज्ञ अपनी सिफ़ारिश वापस लेता है क्योंकि दूरदर्शी ने उसे मना लिया। संशयवादी फ़ोकसर की दलील को और मज़बूत करता है। यह किसी एक प्रॉम्प्ट से संभव नहीं है।

**इंटरव्यू।** बोर्ड बुलाने से पहले हैटर स्पष्टीकरण के सवाल पूछता है। एक अधूरे सवाल पर सात जवाब मिलने का कोई फ़ायदा नहीं।

**संदर्भ।** पहली बार चलाने पर Claude आपकी स्थिति के बारे में पूछता है और उसे सहेज लेता है। हर सलाहकार को यह संदर्भ अपने आप मिलता है — हर बार यह बताने की ज़रूरत नहीं कि आप कौन हैं।

**संश्लेषण ≠ औसत।** हैटर समझौता नहीं खोजता। वह असहमतियों का नक्शा बनाता है: कहाँ सहमति बनी, कहाँ मतभेद रहे, क्यों — और फिर एक ऐसा विचार जोड़ता है जो पूरी स्थिति को उलट कर दिखाता है।

## उदाहरण प्रश्न

Wonderboard सिर्फ़ बिज़नेस तक सीमित नहीं है। यह किसी भी ऐसे फ़ैसले के लिए है जिसमें कई नज़रियों की ज़रूरत हो:

🏢 **रणनीति।** "मेरे पास 5 प्रॉडक्ट हैं और एक इंसान। किसे बंद करूँ, किसे रखूँ?"
🚀 **लॉन्च।** "मैं अमेरिकी बाज़ार में उतरना चाहता हूँ एक ऐसे प्रॉडक्ट के साथ जो सिर्फ़ मेरे देश में लोकप्रिय है।"
💰 **पैसा।** "निवेशक का ऑफ़र आया है। लूँ या बूटस्ट्रैप रहूँ?"
🧠 **करियर।** "उम्र 35, नौकरी से थक गया। अकेले शुरू करूँ या नहीं?"
🏗️ **आर्किटेक्चर।** "3 साल के प्रोजेक्ट के लिए मोनोलिथ या माइक्रोसर्विसेज़?"
📝 **कंटेंट।** "30,000 लेखों वाला पोर्टल है। प्लेटफ़ॉर्म माइग्रेशन में इसे कैसे बचाऊँ?"
🤝 **बातचीत।** "क्लाइंट 40% डिस्काउंट माँग रहा है। क्या करूँ?"

## इंस्टॉल करें

यह Claude Code में पेस्ट करें:

```
Clone https://github.com/aveleazer/wonderboard and run install.sh
```

Claude इसे सेट अप करेगा, आपकी स्थिति पूछेगा, और Wonderboard ब्राउज़र में खोलेगा। अगली बार — `/wonderboard` या बस Claude से बोर्ड बुलाने को कहें।

**आवश्यकताएँ:** Node.js 18+, Claude Code CLI (आपकी मौजूदा सब्सक्रिप्शन पर काम करता है — कोई API key नहीं चाहिए)।

### टोकन

Wonderboard काफ़ी टोकन ख़र्च करता है। Opus पर एक पूरा सेशन: 7 सलाहकार × 2 राउंड + इंटरव्यू + संश्लेषण = 16+ Claude कॉल, 3,00,000+ टोकन।

बिना टोकन ख़र्च किए इंटरफ़ेस देखने के लिए टेस्ट मोड इस्तेमाल करें (`test question` टाइप करें)।

## बोर्ड

| | सलाहकार | नज़रिया |
|---|---|---|
| 🎯 | फ़ोकसर | "आधा फेंक दो। बचा क्या?" |
| ⚔️ | रणनीतिकार | "कहाँ लड़ना है, कहाँ पीछे हटना है?" |
| 💰 | व्यावहारिक | "यूनिट इकॉनॉमिक्स दिखाओ। खाई कहाँ है?" |
| 🔥 | संशयवादी | "यह कहाँ टूटेगा? ग़लती की क़ीमत कौन चुकाएगा?" |
| 🔧 | प्रॉडक्ट वाला | "6 हफ़्तों में क्या शिप कर सकते हो?" |
| 📢 | मार्केटर | "इसके बारे में कौन दोस्त को बताएगा और क्यों?" |
| 🔮 | दूरदर्शी | "10 साल बाद यह कैसा दिखेगा?" |
| 🎩 | हैटर | इंटरव्यू करता है, संश्लेषण करता है, पासा पलटता है |

## सेशन कैसे चलता है

1. **आपका सवाल** — जिस विषय पर सलाह चाहिए
2. **हैटर इंटरव्यू** — संदर्भ निकालने के लिए स्पष्टीकरण के सवाल
3. **राउंड 1** — हर सेज स्वतंत्र रूप से जवाब देता है + अपना एक अनुवर्ती सवाल पूछता है
4. **अनुवर्ती प्रश्नावली** — हैटर सेज के सवालों को एक केंद्रित सर्वे में संकलित करता है
5. **राउंड 2** — सेज एक-दूसरे के जवाब + आपके जवाब पढ़ते हैं, कार्य-योजना के साथ अंतिम राय देते हैं
6. **संश्लेषण** — हैटर सारांश देता है: सहमति, मतभेद, और एक ऐसा विचार जो सब उलट दे

## कॉन्फ़िगरेशन

- `context.md` — आपकी स्थिति (पहली बार चलाने पर Claude पूछता है और भरता है, gitignored)
- `profiles/*.md` — सलाहकार व्यक्तित्व (पूरी तरह संपादन-योग्य, अपने भी जोड़ सकते हैं)
- `prompts/*.md` — प्रॉम्प्ट टेम्पलेट
- 10 UI भाषाएँ, ब्राउज़र से स्वतः पहचान
- डार्क मोड — क्योंकि बड़े फ़ैसले रात को ही सबसे अच्छे होते हैं

## आर्किटेक्चर

शून्य डिपेंडेंसी। सिर्फ़ Node.js के बिल्ट-इन मॉड्यूल।

| फ़ाइल | क्या करती है |
|---|---|
| `server.js` | HTTP सर्वर, CLI के ज़रिए Claude को कॉल, सेशन लॉजिक |
| `ui.html` | सिंगल-पेज इंटरफ़ेस — कोई फ़्रेमवर्क नहीं, कोई बिल्ड स्टेप नहीं |
| `profiles/` | व्यक्तित्व सिस्टम प्रॉम्प्ट |
| `prompts/` | प्रॉम्प्ट टेम्पलेट |

डिज़ाइन दर्शन के लिए देखें [PRINCIPLES.md](PRINCIPLES.md)।

## टेस्ट मोड

पूरा सेशन मॉक डेटा से चलाने के लिए `test question` टाइप करें। कोई API कॉल नहीं, कोई टोकन ख़र्च नहीं। इंटरफ़ेस जानने के लिए बढ़िया तरीक़ा।
