# Wonderboard

7 位 AI 顾问，专门跟你唱反调，也互相拆台。

> *"唯一一个敢说你不爱听的话的工具。"*

## 问题所在

你向 AI 提了个问题，得到一个回答——礼貌、中庸、四平八稳。没有人会说"这事会搞砸"，没有人会问"你为什么非得做这个"，也没有人会提出那个你从没想过的疯狂方案。

现实中，最好的决策来自于想法迥异的人之间的争论。Wonderboard 就是制造这种争论的。

## 跟直接问 Claude 有什么不同

**隔离。** 每位顾问独立思考，各自调用一次 Claude。当一条提示词试图同时扮演怀疑派和远见者——你只会得到一团浆糊。隔离之后——你才能得到真正的分歧。

**两轮对话。** 第一轮，顾问们互不知晓彼此的观点各自作答。第二轮，他们读完所有人的回答后争论、改变立场。实用派因为被远见者说服而收回了建议，怀疑派强化了聚焦者的论点。一条提示词做不到这些。

**访谈。** 疯帽匠会先问你一些澄清问题，然后才召集顾问团。你不会拿到七个针对模糊问题的答案。

**上下文。** 首次启动时，Claude 会了解你的情况并保存下来。之后每位顾问都会自动获得这些上下文——不用每次重新解释你是谁。

**综合 ≠ 取平均值。** 疯帽匠不追求折中。他梳理分歧：谁达成了共识、谁产生了对立、为什么——然后补上一个垂直于所有观点的想法，翻转整个局面。

## 适用场景

Wonderboard 不只是商业工具。任何需要多元视角的决策都适用：

🏢 **战略。** "手上 5 个产品就 1 个人，砍哪些、留哪些？"

🚀 **出海。** "产品只在国内火，想进美国市场。"

💰 **融资。** "收到了投资意向，要不要拿？还是继续自力更生？"

🧠 **职业。** "35 岁，上班上够了。出来单干靠谱吗？"

🏗️ **架构。** "一个三年期项目，选单体还是微服务？"

📝 **内容。** "手上有个 3 万篇文章的站点，平台迁移怎么才不翻车？"

🤝 **谈判。** "客户开口就要打六折，怎么办？"

## 安装

在 Claude Code 中粘贴以下内容：

```
Clone https://github.com/aveleazer/wonderboard and run install.sh
```

Claude 会自动完成安装，了解你的情况，然后在浏览器中打开 Wonderboard。之后直接 `/wonderboard` 或者让 Claude 召集顾问团即可。

**环境要求：** Node.js 18+，Claude Code CLI（使用你现有的订阅，无需 API 密钥）。

### 令牌消耗

Wonderboard 比较消耗令牌。一次完整的 Opus 会话：7 位顾问 × 2 轮 + 访谈 + 综合 = 16 次以上 Claude 调用，30 万以上令牌。

想体验界面又不想花令牌？使用测试模式（输入 `test question`）。

## 顾问团

| | 顾问 | 视角 |
|---|---|---|
| 🎯 | 聚焦者 | "砍掉一半，还剩什么？" |
| ⚔️ | 战略家 | "哪里进攻，哪里撤退？" |
| 💰 | 实用派 | "单位经济模型拿来看看，护城河在哪？" |
| 🔥 | 怀疑派 | "哪里会出问题？谁来买单？" |
| 🔧 | 产品人 | "六周内能交付什么？" |
| 📢 | 营销者 | "谁会把这个安利给朋友？为什么？" |
| 🔮 | 远见者 | "十年后这件事会变成什么样？" |
| 🎩 | 疯帽匠 | 提问、综合、颠覆全场 |

## 会话流程

1. **你的问题** — 你需要什么建议
2. **疯帽匠访谈** — 通过澄清问题提取上下文
3. **第一轮** — 每位顾问独立作答，并提出自己的追问
4. **追问问卷** — 疯帽匠将顾问们的问题汇编成一份精简问卷
5. **第二轮** — 顾问们阅读彼此的回答和你的补充，给出最终立场和行动方案
6. **综合** — 疯帽匠总结：共识、分歧，以及一个颠覆性的垂直观点

## 配置

- `context.md` — 你的情况（首次启动时 Claude 会询问并填写，已加入 gitignore）
- `profiles/*.md` — 顾问人设（完全可编辑，也可以添加自己的）
- `prompts/*.md` — 提示词模板
- 10 种界面语言，根据浏览器自动检测
- 深色模式——因为重要的决定最适合在深夜做出

## 架构

零依赖。仅使用 Node.js 内置模块。

| 文件 | 功能 |
|---|---|
| `server.js` | HTTP 服务器，通过 CLI 调用 Claude，会话逻辑 |
| `ui.html` | 单页界面——无框架，无构建步骤 |
| `profiles/` | 顾问人设的系统提示词 |
| `prompts/` | 提示词模板 |

设计理念详见 [PRINCIPLES.md](PRINCIPLES.md)。

## 测试模式

输入 `test question` 即可用模拟数据运行完整会话。不调用 API，不消耗令牌。非常适合体验界面。
