# Wonderboard

7 ИИ-советников, которые спорят с вами и друг с другом.

> *«Единственный инструмент, который скажет то, что вы не хотите слышать.»*

## Проблема

Вы задаёте вопрос ИИ. Получаете один ответ. Вежливый, сбалансированный, усреднённый. Никто не скажет «это провалится». Никто не спросит «а зачем ты вообще это делаешь?». Никто не предложит безумную альтернативу, о которой вы не думали.

В реальной жизни лучшие решения рождаются из спора между людьми, которые думают по-разному. Wonderboard создаёт такой спор.

## Пример сессии

Вопрос: *«Хочу расширить линейку продуктов — какие ещё AI-советы будут полезны пользователям?»*

**🎯 Фокусировщик:** «Ты расширяешь линейку продукта, которым пользуешься только ты. Это не стратегия. Это прокрастинация в красивой упаковке.»

**🔮 Визионер:** «Вы спрашиваете "какие ещё советы добавить?" — но это вопрос архитектора витрины, не архитектора системы. Это как Энгельбарт в 1968 году спрашивал бы "какие ещё команды добавить в командную строку?" — вместо того чтобы изобрести мышь.»

**💰 Прагматик (раунд 1):** «Шляпник подбирает состав под вопрос — это ваш потенциальный ров.»

**💰 Прагматик (раунд 2, прочитав Визионера):** «Беру это назад. Это протокол, не фича. Но для вас сейчас — преждевременная абстракция.»

**🎩 Шляпник (синтез):** «Board конкурирует не с консультантами. Он конкурирует с тишиной.»

## Чем это отличается от «просто спросить Claude»

**Изоляция.** Каждый советник думает независимо, в своём собственном вызове API. Когда один промт пытается быть одновременно скептиком и визионером — получается каша. Когда они изолированы — получается настоящее несогласие.

**Два раунда.** В первом раунде советники отвечают, не зная мнений друг друга. Во втором — читают все ответы, спорят и меняют позиции. Прагматик берёт назад рекомендацию, потому что Визионер его убедил. Скептик усиливает аргумент Фокусировщика. Это невозможно получить в одном промте.

**Интервью.** Шляпник задаёт уточняющие вопросы перед тем, как созвать Совет. Вы не получите семь ответов на неправильно поставленный вопрос.

**Контекст.** Файл `context.md` содержит описание вашего бизнеса. Каждый советник получает его автоматически — не нужно каждый раз объяснять, кто вы и чем занимаетесь.

**Синтез ≠ усреднение.** Шляпник не ищет компромисс. Он строит карту разногласий: где сошлись, где разошлись, почему — и добавляет одну перпендикулярную мысль, переворачивающую ситуацию.

## Примеры вопросов

Wonderboard не ограничен бизнесом. Это инструмент для любого решения, где нужна полифония:

🏢 **Стратегия.** «У меня 5 продуктов, один человек. Что убить, что оставить?»

🚀 **Запуск.** «Я хочу выйти на западный рынок с продуктом, который популярен только в России.»

💰 **Деньги.** «Мне предложили инвестора. Брать или оставаться bootstrapped?»

🧠 **Карьера.** «Мне 35, я устал от найма. Уходить в своё дело или нет?»

🏗️ **Архитектура.** «Монолит или микросервисы для проекта на 3 года?»

📝 **Контент.** «У меня портал с 30K статей. Как его не убить при переезде на новый движок?»

🤝 **Переговоры.** «Клиент хочет скидку 40%. Что делать?»

## Установка

Вставьте это в [Claude Code CLI](https://docs.anthropic.com/en/docs/claude-code):

> Склонируй https://github.com/aveleazer/wonderboard и запусти install.sh

Claude установит и откроет Wonderboard в браузере. В следующий раз — просто `/wonderboard` или попросите Claude созвать совет.

**Требования:** Node.js 18+, [Claude Code CLI](https://docs.anthropic.com/en/docs/claude-code) (работает на вашей подписке, API-ключи не нужны).

### Токены

Wonderboard — требовательный инструмент. Одна полная сессия: 7 советников × 2 раунда + интервью + синтез = 16+ вызовов Claude. На Claude Max/Pro подписке это входит в лимит.

Для знакомства с интерфейсом используйте тестовый режим (введите `test question`) — он работает без вызовов API.

| ⚔️ | Стратег | «Где сражаться, а где отступить?» |
| 💰 | Прагматик | «Покажи юнит-экономику. Где ров?» |

| 🔮 | Визионер | «Как это выглядит через 10 лет?» |
| 🎩 | Шляпник | Интервьюирует, синтезирует, переворачивает |

## Как проходит сессия

1. **Ваш вопрос** — о чём хотите получить совет
2. **Интервью Шляпника** — уточняющие вопросы для извлечения контекста
3. **Раунд 1** — каждый Мудрец отвечает независимо + задаёт свой уточняющий вопрос
4. **Дополнительный опросник** — Шляпник компилирует вопросы Мудрецов в фокусированную анкету
5. **Раунд 2** — Мудрецы читают ответы друг друга + ваши ответы, выдают финальные позиции с планами действий
6. **Синтез** — Шляпник подводит итог: консенсус, разногласия и одна перпендикулярная мысль

## Настройка

- `context.md` — ваш бизнес-контекст (создаётся автоматически, в git не попадает)
- `profiles/*.md` — персоны советников (можно редактировать и добавлять свои)
- `prompts/*.md` — шаблоны промптов
- 10 языков UI, определяются автоматически из браузера
- Тёмная тема — потому что важные решения лучше принимать ночью

## Архитектура

Ноль зависимостей. Только встроенные модули Node.js.

| Файл | Что делает |
|---|---|
| `server.js` | HTTP-сервер, вызовы Claude через CLI, логика сессий |
| `ui.html` | Одностраничный интерфейс — без фреймворков, без сборки |
| `profiles/` | Системные промпты персон |
| `prompts/` | Шаблоны промптов |

Философия проекта — в [PRINCIPLES.md](PRINCIPLES.md).

## Тестовый режим

Введите `test question` в поле вопроса, чтобы запустить полную сессию с мок-данными. Без вызовов API, без затрат токенов. Удобно для знакомства с интерфейсом.
