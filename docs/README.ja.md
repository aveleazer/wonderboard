# Wonderboard

あなたにも、互いにも、遠慮なく異を唱える7人のAIアドバイザー。

> *「聞きたくないことを、はっきり言ってくれる唯一のツール。」*

## 課題

AIに質問する。答えがひとつ返ってくる。丁寧で、バランスが取れていて、無難にまとまっている。「これは失敗する」と言う者はいない。「そもそもなぜそれをやるのか」と問う者もいない。自分では思いつかなかった突拍子もない選択肢を提示する者もいない。

現実世界では、最良の意思決定は異なる考え方を持つ人々の議論から生まれる。Wonderboardはその議論を再現する。

## 普通にClaudeに聞くのと何が違うのか

**分離。** 各アドバイザーは独立したClaude呼び出しで、個別に思考する。ひとつのプロンプトで懐疑派と先見者を同時に演じさせると、中途半端な回答になる。分離すれば、本物の意見対立が生まれる。

**2ラウンド制。** ラウンド1では、各アドバイザーは他のメンバーの意見を見ずに回答する。ラウンド2では、全員の回答を読み、議論し、立場を変える。先見者に説得されて実務家が推奨を撤回する。懐疑派が集中の達人の論点を補強する。これはひとつのプロンプトでは実現できない。

**インタビュー。** 帽子屋がボードを招集する前に、あなたに掘り下げた質問をする。曖昧な質問に対して7つの回答が返ってくることはない。

**コンテキスト。** 初回起動時にClaudeがあなたの状況を聞き取り、保存する。以降、すべてのアドバイザーにそのコンテキストが自動的に渡される。毎回自己紹介をやり直す必要はない。

**統合 ≠ 平均化。** 帽子屋は妥協点を探さない。意見の対立を地図のように整理する——どこで一致し、どこで分かれ、なぜそうなったか。そして、状況をひっくり返すような斜め上の一言を加える。

## 質問の例

Wonderboardはビジネスに限らない。多角的な視点が必要なあらゆる意思決定に使えるツールだ。

🏢 **戦略。**「プロダクトが5つあって、人手は1人。何を切って、何を残すべきか？」
🚀 **市場参入。**「自国でしか売れていないプロダクトで、アメリカ市場に挑みたい。」
💰 **資金調達。**「投資家からオファーが来た。受けるか、自力でやり続けるか？」
🧠 **キャリア。**「35歳、会社勤めに疲れた。独立すべきか？」
🏗️ **アーキテクチャ。**「3年計画のプロジェクト。モノリスかマイクロサービスか？」
📝 **コンテンツ。**「3万記事のポータルサイトがある。プラットフォーム移行で殺さないためには？」
🤝 **交渉。**「クライアントが40%の値引きを要求してきた。どうすべきか？」

## インストール

Claude Codeに以下を貼り付ける：

```
Clone https://github.com/aveleazer/wonderboard and run install.sh
```

Claudeがセットアップを行い、あなたの状況をヒアリングし、ブラウザでWonderboardを開く。次回以降は `/wonderboard` と入力するか、Claudeにボードの招集を頼むだけでいい。

**必要環境：** Node.js 18+、Claude Code CLI（既存のサブスクリプションで動作——APIキーは不要）。

### トークン

Wonderboardはトークンを大量に消費する。Opusでのフルセッション：7アドバイザー × 2ラウンド + インタビュー + 統合 = 16回以上のClaude呼び出し、30万トークン超。

トークンを使わずにインターフェースを試すには、テストモードを使う（`test question` と入力）。

## ボードメンバー

| | アドバイザー | 視点 |
|---|---|---|
| 🎯 | 集中の達人 | 「半分捨てろ。残ったものは何だ？」 |
| ⚔️ | 戦略家 | 「どこで戦い、どこで引くか？」 |
| 💰 | 実務家 | 「ユニットエコノミクスを見せろ。堀はどこだ？」 |
| 🔥 | 懐疑派 | 「どこで破綻する？ミスの代償は誰が払う？」 |
| 🔧 | プロダクト屋 | 「6週間で出せるものは何だ？」 |
| 📢 | マーケター | 「誰が、なぜ、これを人に勧める？」 |
| 🔮 | 先見者 | 「10年後、これはどうなっている？」 |
| 🎩 | 帽子屋 | インタビュー、統合、そしてちゃぶ台返し |

## セッションの流れ

1. **あなたの質問** —— アドバイスが欲しいテーマ
2. **帽子屋のインタビュー** —— コンテキストを引き出すための掘り下げ質問
3. **ラウンド1** —— 各賢者が独立して回答 + 独自のフォローアップ質問
4. **フォローアップ質問票** —— 帽子屋が賢者たちの質問を集約した追加アンケート
5. **ラウンド2** —— 賢者たちが互いの回答とあなたの回答を読み、アクションプラン付きの最終見解を提示
6. **統合** —— 帽子屋が総括：合意点、対立点、そして状況をひっくり返す一言

## 設定

- `context.md` —— あなたの状況（初回起動時にClaudeがヒアリングして記入、gitignore対象）
- `profiles/*.md` —— アドバイザーのペルソナ（自由に編集可能、独自の追加も可能）
- `prompts/*.md` —— プロンプトテンプレート
- UIは10言語対応、ブラウザの設定から自動検出
- ダークモード —— 重要な決断は夜にこそ下すものだから

## アーキテクチャ

依存パッケージはゼロ。Node.js組み込みモジュールのみ。

| ファイル | 役割 |
|---|---|
| `server.js` | HTTPサーバー、CLI経由でClaudeを呼び出し、セッションロジック |
| `ui.html` | シングルページのインターフェース —— フレームワークなし、ビルドステップなし |
| `profiles/` | ペルソナのシステムプロンプト |
| `prompts/` | プロンプトテンプレート |

設計思想は [PRINCIPLES.md](PRINCIPLES.md) を参照。

## テストモード

`test question` と入力すると、モックデータでフルセッションが実行される。API呼び出しなし、トークン消費なし。インターフェースの探索に最適。
